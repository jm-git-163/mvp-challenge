# CHECKLIST — 카메라 전/후면 전환 (CAMERA-SWAP, 2026-04-23)

자동화 범위 외. 사용자가 실기기에서 아래 항목을 육안으로 검증해야 한다.

## 대상 기기
- [ ] iPhone Safari (iOS 17+)
- [ ] Android Chrome (최신)

## 핵심 동작 검증
- [ ] 녹화 시작 후 🔄 버튼 활성화 (녹화 전/중 모두 누를 수 있음)
- [ ] 카운트다운(3-2-1) 중에는 🔄 비활성화
- [ ] 녹화 중 전→후면 토글 시 **녹화본이 중간에 끊기지 않음** (결과 mp4 확인)
- [ ] 녹화 중 후→전면 토글 시에도 녹화본 연속
- [ ] 전환 중 캔버스에 "📷 전환 중…" 오버레이 표시 (약 200~600ms)
- [ ] 전환 완료 후 1초 이내 새 카메라 영상이 표시됨
- [ ] 후면일 때 화면 mirror 되지 않음 (글자·텍스트가 정상 방향)
- [ ] 전면일 때 화면 mirror 됨 (셀피 모드)

## 쿨다운 / 중복 차단
- [ ] 🔄 연타 시 1.5초 안에 두 번째 전환은 무시됨
- [ ] 전환 중에 다시 누르면 무시 (버튼도 disabled)

## UI 라벨
- [ ] 🔄 버튼 옆에 현재 facing 라벨 "전면" 또는 "후면" 항상 표시

## 실패 경로
- [ ] 후면 카메라가 없는 단말(데스크톱 웹캠 등)에서 🔄 누르면
      2.5초간 "카메라 전환 실패 — 다시 시도해주세요" 토스트 표시
- [ ] 실패 후에도 기존 카메라는 정상 동작 계속 (녹화 끊기지 않음)

## cameraPlan 토스트 (food-review 템플릿)
- [ ] `food-review` 템플릿으로 녹화 시작
- [ ] 0~5s 구간 — 전면 카메라 권장 (별도 토스트 없음)
- [ ] 약 4초 시점에 상단 노란 토스트 "5초 후 후면 전환 — 음식 클로즈업" 표시
- [ ] 토스트 확인 후 🔄 누르면 후면 카메라로 전환
- [ ] 약 11초 시점에 "5초 후 전면 전환 — 먹방 리액션" 토스트 표시
- [ ] 자동 전환은 **아님** — 유저가 직접 🔄 를 눌러야 바뀜

## 회귀
- [ ] 🔄 를 한 번도 누르지 않은 녹화 시나리오는 기존과 동일
- [ ] 다른 템플릿(cameraPlan 없음)은 토스트 없이 기존처럼 동작
- [ ] 녹화 종료 후 결과 화면에서 mp4 정상 재생
